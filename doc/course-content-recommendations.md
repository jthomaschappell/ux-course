# Course Content Recommendations

Summary of recommendations made after scanning the 7-Day UX Fluency Plan content (days data, artifact forms, reference panel, types). Use this as a checklist for future edits.

---

## 1. Fill study gaps

- Add **study items** for Days 2, 4, 6, and 7 (they had empty `studyItems`).
- Examples: Day 2 → critique script, replacing vague language, 2-min pitch; Day 4 → moderator script, task design, severity; Day 6 → case study structure, before/after, KPIs; Day 7 → answer structure, pushback patterns, portfolio framing.

---

## 2. Clarify Day 1

- Make critique targets **product-agnostic** (e.g. “A productivity tool you use”, “An app you’re building or own”) instead of “DrawFi or similar”.
- Turn **“No watching without writing”** into a short guideline: use the product in small chunks and complete each section before moving on.

---

## 3. Course intro and per-day context

- Add a **course intro**: who it’s for, how to use the app, and rough time (e.g. 1–2 hours per day).
- Add **per-day** optional: **objective** (why this day), **estimated minutes**, and **one thing to nail today**.

---

## 4. Reference panel

- Add **URLs** to reference resources and make names clickable.
- Add **severity definitions** for usability findings (High / Medium / Low).
- Add a **one-line mental model** (e.g. Goal → Context → Friction → … → Outcome).
- **Map “Goal by Day 7”** to the days that support each goal (e.g. “Days 1–3”, “Day 4”).

---

## 5. Scaffold artifact forms

- **Day 1:** Placeholder examples for Primary User Goal and Expected Impact.
- **Day 3:** “Name the screen or product” and example-style placeholders.
- **Day 4:** Add **“Task given to participant”** and a short script note (don’t ask “Would you use this?”).
- **Day 5/6:** Mention **view-only Figma link** in instructions/placeholders.
- **Day 7:** Show **pushback prompts** next to the textarea and use a single **interview formula** (e.g. from Reference).

---

## 6. Copy and “Good looks like”

- **Day 5:** Add “Use auto-layout and components” and “Share a view-only Figma link”.
- **Day 6:** Add **“Good looks like”** (e.g. clear before/after, one KPI, redesign link).
- **Day 1:** Add **“Good looks like”** (e.g. at least one heuristic named NNG-style, testable hypothesis).
- **Outcome:** Add “This plan builds fluency, not full expertise.”

---

## 7. Day 4 usability testing

- In the script: **don’t ask preference questions**; focus on observed behavior and task completion.
- In the deliverable/Reference: **define severity** (High = blocks task; Medium = major friction; Low = annoyance).

---

## 8. Optional

- **Time estimates** per day (or per task) so learners can plan.
- **Rubrics or success criteria** for 1–2 high-stakes days (e.g. Day 1, Day 6).
